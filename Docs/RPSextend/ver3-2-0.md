# ver3.2.0 — 切断時の自動終了・ライト水色テーマ・ルール説明画面

- **バージョン**: ver3.2.0
- **日付**: 2026-07-11
- **種別**: 実装（ルール変更なし。切断処理の改善・UI 配色の刷新・ルール説明追加）
- **対象**: `Online/web_server.py`, `Online/engine.py`

## 概要

ブラウザGUI版で、
1. **対戦中に片方が切断したら、もう片方も含めて対戦を自動終了**するようにした。
2. 全体の配色を**黒基調 → 水色（みずいろ）基調のライトテーマ**に刷新した
   （`home.sumyuu.com` のパステル水色イメージ）。
3. **初心者向けのルール説明画面**を追加した（画面右上「📖 ルール説明」）。

ゲームのルール・判定ロジック（ver2.5.0）は不変。

## 変更点

### 1. 切断時の自動終了

**修正前の問題**: 切断検知（reaper）は、切断したプレイヤー自身のキューにのみ
`DISCONNECT` を投入していた。そのため「相手の手番中」に切断されると、対戦進行
（`Match`）は生存プレイヤーの入力待ちでブロックしたまま、切断シグナルが読まれず、
**生存プレイヤーが延々と待たされる**状態になり得た。

**修正**:
- `engine.Session.await_choice`: キューに `Disconnected` インスタンスが投入された場合、
  それを（切断者名を保持したまま）送出するように変更。どちらの手番待ちでも中断できる。
- `web_server.py`: 切断時に**対戦相手を含む両者のキュー**へ切断シグナルを送る
  `_abort_match()` を追加。共通の `match` ID で相手を特定する。
- reaper と離脱通知を共通化する `_drop_session()` を追加。切断検知のしきい値は
  `DISCONNECT_TIMEOUT=15` 秒、監視間隔を 3 秒に短縮（反応を速く）。
- **`POST /leave`** を追加。ブラウザ側は `pagehide` で `navigator.sendBeacon('/leave')`
  を送り、**タブを閉じた瞬間**に reaper のタイムアウトを待たず対戦を終了できる。

いずれの場合も、生存プレイヤーには
`相手(名前)が切断しました。対戦を終了します。` の GAMEOVER が届く。

### 2. 水色基調のライトテーマ

- CSS 変数を刷新（`--bg` 水色グラデーション、`--panel` 白、`--accent` シアン `#00c2d6` 等）。
- 背景は水色の縦グラデーション、カードは白＋淡いシアンのグロー影。
- 盤面・ログ・入力欄・勝敗バナー（win/lose/draw）をライト配色に調整。
  RPSカードの色分け（赤/緑/青）は白背景でも視認できる枠色に。

### 3. ルール説明画面（初心者向け）

- ヘッダーに「📖 ルール説明」ボタンを追加し、モーダルで遊び方を表示。
  参加前・対戦中どちらからでも開ける（× / 背景クリック / Esc で閉じる）。
- 内容は README / 完成仕様（latest.md, ver2.5.0）の要点を**初めての人向けに平易化**：
  目的・使う3種のカード・「封印≠本命」の要点・1ターンの流れ・点の入り方・
  サドンデス・立ち回りのコツ。

## 変わっていないもの
- ルール（ver2.5.0）、秘匿方式、サーバー一元管理。
- 既存エンドポイント `/`, `/join`, `/poll`, `/choice`, `/admin/status` の挙動。
- CUI 版（`server.py` + `client.py`）。

## 動作検証
- **切断終了**: 2人をマッチ後、手番でない側を `POST /leave` で離脱させ、
  生存側のポーリングで `GAMEOVER 相手(bob)が切断しました…` を受信することを確認
  （従来ハングしていたケース）。
- **テーマ**: ブラウザプレビューで参加画面・対戦画面（スコアボード/盤面/ログ/
  カードボタン/切断バナー）を目視確認。コンソールエラーなし。
- **ルール説明**: モーダルの開閉（ボタン/×/Esc）と全文表示を目視確認。
