# ver3.1.1 — 管理用ステータス確認エンドポイントの追加

- **バージョン**: ver3.1.1
- **日付**: 2026-07-10
- **種別**: 実装（ルール変更なし。運用・管理機能の追加）
- **対象**: `Online/web_server.py`（`Online/README.md` に運用手順を追記）

## 概要

本番運用（OCI + Nginx + HTTPS 常駐）で**現在接続中のプレイヤーを確認**できるよう、
Web GUI サーバーに管理用エンドポイント **`GET /admin/status`** を追加した。
ゲームのルール・判定ロジック（ver2.5.0）には一切変更なし。

## 変更点

### `GET /admin/status`（現在のプレイヤー確認）
- 接続中セッションを JSON で返す。
  - `counts`: 合計 / 待機中(`waiting`) / 対戦中(`playing`) / 切断待ち(`dead`) の人数。
  - `matches`: 対戦中のペア（同じ `match` ID の2人が対戦相手）。
  - `waiting`: マッチ待ちのプレイヤー。
  - 各プレイヤー: `name` / 短縮 `id`(token先頭8桁) / `idle_sec`(最終ポーリングからの秒) / `age_sec`(参加からの秒) / `dead`。
- マッチ成立時に対戦ペアへ共通の `match` ID を付与（`_start_match_if_ready`）。
  セッションレコードに `match` / `joined` を追加。

### アクセス制御（既定で安全）
- 環境変数 `RPS_ADMIN_TOKEN` **未設定**時: サーバー内からの **loopback 直叩き
  (`127.0.0.1:8000`) のみ許可**。Nginx 経由（公開URL）は `X-Forwarded-For` が付くため
  `403 Forbidden`。
- `RPS_ADMIN_TOKEN` **設定**時: `?token=` が一致した場合のみ許可（公開URLからも確認可）。
- トークンは個人情報同様に扱い、リポジトリに直書きしない。

## 変わっていないもの
- ルール（ver2.5.0）・秘匿方式・サーバー一元管理・既存エンドポイント
  （`/`, `/join`, `/poll`, `/choice`）の挙動。
- CUI 版（`server.py` + `client.py`）。

## 動作検証（ローカル）
- 3人参加（2人マッチ + 1人待機）で `/admin/status` が
  `matches`(ペア) と `waiting`(単独) を正しく分類することを確認。
- loopback 直叩き → 200、`X-Forwarded-For` 付き（Nginx 相当）でトークン未設定 → 403、
  トークン設定時は正しい `?token=` のみ 200・誤り/未指定は 403 を確認。

## 使い方（要約）
```bash
# サーバー上（SSHログイン後）: 内部ポートを直接叩く
curl -s http://127.0.0.1:8000/admin/status | python3 -m json.tool

# 公開URLから見たい場合のみ RPS_ADMIN_TOKEN を設定し、token を付与
curl -s "https://<ドメイン>/admin/status?token=<設定値>" | python3 -m json.tool
```
