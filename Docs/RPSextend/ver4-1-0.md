# ver4.1.0 — 選択操作もカードクリックに（プロンプトのカードUI化）

- **バージョン**: ver4.1.0
- **日付**: 2026-07-12
- **種別**: 実装（ルール変更なし。ブラウザGUI版の入力UIを刷新）
- **対象**: `Online/web_server.py`, `Online/engine.py`

## 概要

ver4.0.0 で盤面をカードUI化したのに続き、**プレイヤーの選択操作（封印・効果・能力・
本命・guess）も、テキストボタンから「カード画像をクリックして選ぶ」形式**に変更した。
盤面のミニカードと同じ見た目のカードが選択肢として並び、クリック（またはEnter/Space）で
選べる。

ゲームのルール・判定ロジック（ver2.5.0）は不変。

## 変更点

### 1. サーバー: プロンプトに選択肢の「種別(kind)」を付与（`engine.py`）

- `Session.prompt(pid, text, options, kind=None)` に `kind` を追加し、`PROMPT` メッセージへ
  同梱。GUI が適切なカードで描画できるようにする。種別は次の5つ:
  - `rps`（封印／本命／覗き見の開示指定／サドンデス）
  - `effect`（効果カード）
  - `ability`（能力カード）
  - `hand`（guess の「手」予想）
  - `color`（guess の「色」予想）
- `_rps_options()` を拡張し、各選択肢に `hand`/`color` を持たせて GUI が RPS ミニカードを
  描画できるようにした。
- `prompt_both` / `prompt_one` と各フェイズの呼び出しを `kind` を渡すように更新
  （`specs` を 4 要素タプル `(pid, text, options, kind)` に）。

### 2. クライアント: 選択肢をクリック可能なカードで描画（`web_server.py`）

- `showPrompt()` を刷新。`msg.kind` に応じて選択肢を盤面と同じミニカードで描画する
  `optionCardHTML()` を追加:
  - `rps` → 色枠＋絵文字＋手名＋色サークル（R/G/B）
  - `effect` → 金枠＋「◯◯で勝つ」＋`+1`
  - `ability` → スレート枠＋アイコン（👁️🎯🛡️）
  - `hand` → 絵文字＋手名（シアン枠）
  - `color` → 赤/緑/青で塗った色スウォッチ＋R/G/B
- カードは `.pick` 化してホバーで浮き上がり、クリックで選択（`role=button`・`tabIndex`・
  Enter/Space でも決定できるようアクセシビリティ対応）。
- `kind` 不明時は従来のテキストボタンにフォールバック（後方互換）。

## 変わっていないもの
- ルール（ver2.5.0）、勝敗・guess/guard・サドンデスの判定ロジック。
- 秘匿方式（相手の非公開情報はサーバーが送らない）。
- 既存エンドポイント `/`, `/join`, `/poll`, `/choice`, `/leave`, `/admin/status` の挙動。
- 盤面表示（ver4.0.0）・実況ログ・ルール説明モーダル・切断時の自動終了。
- CUI 版（`server.py` + `client.py`）。

## 動作検証
- ブラウザプレビューで2人をマッチさせ、1ターンの全フェイズを操作して確認:
  - 封印／本命: 手札の RPS カード（色枠＋絵文字＋R/G/B）をクリックして選択。
  - 効果: 金枠「◯◯で勝つ」＋`+1` カードをクリック。
  - 能力: 覗き見／探偵(手)／探偵(手+色)／盾 のスレートカードをクリック。
  - guess(手): 手カード（✊✌️🖐）、guess(色): 赤/緑/青の色スウォッチをクリック。
  - ホバーで浮き上がり、クリックで選択→送信されることを確認。
- コンソールエラーなし。切断時の自動終了バナーも従来どおり表示。
