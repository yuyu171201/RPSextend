# ver4.0.0 — 盤面のビジュアルGUI化（カード画像ベースのミニカード表示）

- **バージョン**: ver4.0.0
- **日付**: 2026-07-12
- **種別**: 実装（ルール変更なし。ブラウザGUI版の盤面表示を全面刷新）
- **対象**: `Online/web_server.py`, `Online/engine.py`

> メジャー番号を 4 に上げたのは、対戦画面の中核である**盤面表示を等幅テキストから
> カードUIへ全面刷新**したため（体験の見た目が大きく変わる区切り）。
> ゲームのルール・判定ロジック（ver2.5.0）は引き続き不変。

## 概要

対戦中の盤面を、これまでの**モノスペース等幅テキスト**表示から、
**ゲームのカード（`cards.html` / `generate_cards.py`）のデザインを模したミニカード**による
**ビジュアル盤面**に作り替えた。ひと目で「誰が何を持っていて・何を使ったか」が分かるようにした。

ゲームのルール・判定ロジック（ver2.5.0）は不変。サーバー一元管理・秘匿方式も不変。

## 変更点

### 1. サーバー: 盤面を構造化データ（`view`）でも送る（`engine.py`）

- `Session.state(lines, view=None)` に第2引数 `view` を追加。
  `STATE` メッセージに GUI 描画用の構造化データを同梱する（`lines` は後方互換で従来通り併送）。
- `Match._build_view(i)` を追加。プレイヤー `i` 視点の盤面を、**秘匿情報を含まない**形で
  辞書化する:
  - `me`: 得点・RPS手札（各札の `hand`/`color` と**開示中フラグ** `revealed`）・
    今ターンの封印/本命（自分のみ中身可視）・効果/能力の手札。
  - `opp`: 得点・**公開済み本命**（表向き）・**非公開枚数**（手札＋封印）・
    開示中の札・**使用済み**効果/能力。
- 相手の手札の中身・封印の中身は一切送らない（従来の秘匿方針を維持）。

### 2. クライアント: カード画像ベースのミニカード描画（`web_server.py`）

- 盤面 `#board` を等幅テキストから、**相手ゾーン**／**自分ゾーン**の2ゾーン構成に変更。
- カードの見た目は印刷用カード（`cards.html`）に準拠したミニカードで表現:
  - **RPSカード**: 手の絵文字（✊✌️🖐）＋手名＋**色枠**（赤/緑/青）＋右上の**色サークル**（R/G/B）。
    直前ターン開示された札には**シアンのグロー枠＋「開示中」バッジ**を付与。
  - **効果カード**: 金枠＋手の絵文字＋「◯◯で勝つ」＋`+1` バッジ。
  - **能力カード**: スレート枠＋アイコン（👁️覗き見／🎯探偵／🛡️盾）＋名称。
  - **非公開札**: 斜めストライプの**カード裏**（`?`）を枚数分並べ、「残り N枚」を併記。
  - 相手の**使用済み**効果/能力は、淡色の丸ピル型チップで表示。
- `renderView(view)` を新設し、`STATE` 受信時に `msg.view` があればそれを描画、
  無ければ従来のテキスト（`lines`）にフォールバック。スコアボード（ターン/名前/得点）も
  `view` から更新する。
- プレイヤー名は `esc()`（HTMLエスケープ）を通して描画（XSS対策）。

## 変わっていないもの
- ルール（ver2.5.0）、勝敗・guess/guard・サドンデスの判定ロジック。
- 秘匿方式（相手の非公開情報はサーバーが送らない）。
- 既存エンドポイント `/`, `/join`, `/poll`, `/choice`, `/leave`, `/admin/status` の挙動。
- 入力（カード風ボタンでの選択）・実況ログ・ルール説明モーダル。
- CUI 版（`server.py` + `client.py`）。

## 動作検証
- ブラウザプレビューで2人をマッチさせ、1ターンを最後まで進行して目視確認:
  - 開始時: 相手ゾーンにカード裏 ×9＋「残り 9枚」、自分の RPS/効果/能力手札がミニカード表示。
  - 覗き見で開示された自札に**「開示中」バッジ＋グロー枠**が付くこと。
  - ターン終了後: 相手の**公開済み本命が表向きカード**で表示され、
    使用済みの効果（「グーで勝つ」）・能力（「覗き見」）が丸チップで表示され、
    非公開枚数が「残り 8枚」に減ること。
  - スコアボードのターン/名前/得点が `view` から正しく更新されること。
- コンソールエラーなし。`.venv/bin/python3.12`（要 Python 3.12）で起動。
