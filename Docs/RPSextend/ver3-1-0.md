# ver3.1.0 — ブラウザGUI版オンライン対戦の追加

- **バージョン**: ver3.1.0
- **日付**: 2026-07-09
- **種別**: 実装（ルール変更なし。ver2.5.0 のルールのままGUIクライアントを追加）
- **対象**: `Online/`（`web_server.py`, `engine.py` 追加、`server.py` リファクタ）

## 概要

ver3.0.0 のCUIオンライン対戦を、**ブラウザで動作するGUI**でも遊べるようにした。
`spec.md` の中央サーバー方式・サーバー一元管理・表示制御による秘匿はそのまま維持し、
クライアントの見た目と操作をGUI化しただけで、ルール・判定ロジックは共通。

## 変更点

### 対戦エンジンの分離（トランスポート非依存化）
- ver3.0.0 では `server.py` に同居していた対戦進行ロジックを **`engine.py`** に分離。
  - `Session`：1プレイヤーとの入出力窓口。送信は `transport.send(obj)`、
    受信（CHOICE等）は `Queue` へ外部から投入される抽象化。
  - `Match`：6ターンのステートマシン・判定（ver3.0.0 と同一挙動）。
- これにより **TCP版（server.py）とWeb版（web_server.py）が同じ `Match` を共用**する。
  ルールの二重実装を避け、片方だけ挙動がずれる事故を防止。

### Web GUI サーバー（`web_server.py`）
- Python標準ライブラリ `http.server`（`ThreadingHTTPServer`）のみで実装。追加依存なし。
- 単一ページ（HTML/CSS/JS を1ファイルに内蔵）をブラウザへ配信。
- ブラウザ ⇔ サーバーは HTTP(JSON)。
  - `POST /join`（名前）→ セッション発行・待機列へ・2人揃えば対戦開始。
  - `GET /poll`（ポーリング）→ サーバーからのメッセージ（HELLO/MSG/STATE/PROMPT/GAMEOVER）を取得。
  - `POST /choice`（PROMPTへの回答）→ 選択をエンジンへ送る。
- 一定時間ポーリングが来ないセッションを**切断扱い**にする reaper を追加（相手へ通知して終了）。

### GUI（ブラウザ画面）
- 参加画面（名前入力）→ 対戦画面（スコアボード / 盤面パネル / 実況ログ / 操作パネル）。
- **PROMPTはカード風ボタン**で提示。RPSカードは色（赤/緑/青）で枠色を分けて視認性を確保。
- 盤面・スコア・ログはポーリングで**リアルタイム更新**。相手の手番中は「相手の操作を待っています…」表示。

## 変わっていないもの
- ルール（ver2.5.0）：封印≠本命、効果公開＋ボーナス、覗き見(floor2)、
  guess(手/手+色, 1〜3T +3)当否のみ、盾、サドンデス（全あいこ引分）。
- 秘匿の方式（暗号なし・表示制御のみ）、サーバー一元管理（チート不可）。
- CUI版（`server.py` + `client.py`）も引き続き利用可能。

## 動作検証
- HTTP経由の自動プレイヤー2体で複数フルゲーム完走（通常決着・サドンデス決着）を確認。サーバー例外なし。
- 実ブラウザ（プレビュー）で参加→封印→効果フェイズへの遷移、カードボタン操作、
  盤面/ログのリアルタイム更新、色分け表示を目視確認。

## 起動方法（要約）
```bash
# GUI版（推奨）
cd Online && python3 web_server.py --port 8000
#   → 各プレイヤーがブラウザで http://<サーバーIP>:8000/ を開く

# CUI版
cd Online && python3 server.py --port 5000
python3 client.py --host <サーバーIP> --port 5000 --name アリス
```
