# RPS Extend - Online Match Specification (オンライン対戦仕様書)

このドキュメントは「ジャンケン・エクステンド」をオンラインで1対1対戦できるようにするための仕様を定義したものです。

---

## 1. 接続方式とUI（システムアーキテクチャ）

本システムは、**CUI（ターミナル上で動作するCLIアプリ）**による対戦を前提とします。

*   **接続方式**: **中央サーバー方式（クライアント・サーバー型）**
*   **通信概要**: 
    *   中央の専用サーバーがゲームホスト（進行・判定役）として稼働します。
    *   両プレイヤーは、SSH（またはTCPソケット）経由でこのサーバーに接続し、ターミナル上で対戦を行います。
*   **状態管理**: 
    *   ゲームのすべての状態（両者の手札、現在の点数、ターンの進行）は中央サーバーが一元管理します。
    *   中央サーバーが情報を管理しているため、プレイヤーがクライアント側で直接チートを行うことはできません。

---

## 2. 情報の秘匿について

このゲームは「お互いに裏向きにカードを出し、同時に公開する」という同時出し（秘匿情報）の要素が含まれますが、通信プロトコルレベルでの厳密な暗号化（ハッシュを用いたコミット＆リビールなど）は行いません。

*   中央サーバーが通信を仲介しているため、プレイヤーがカードを選んだ情報はそのままサーバーに送られます。
*   サーバーは、両方のプレイヤーの選択が揃うまで、その情報をもう一方のプレイヤーの画面（CUI）には「表示しない」という表示上の制御のみで秘匿を実現します。
*   これにより、不要な通信回数の増加を抑え、シンプルにゲームを成立させます。

---

## 3. ターンの進行プロトコル（ステートマシン）

サーバーはゲームの進行状態を管理し、各ターンごとに以下のフェイズ（State）を遷移します。両プレイヤーからの入力が揃った段階でサーバーが処理を行い、結果を両者に通知して次のフェイズへ移行します。

| フェイズ | 内容 |
| :--- | :--- |
| `MATCHING` | 対戦相手の接続待ち。接続完了後、初期手札を配布。 |
| `PHASE_1: DUMMY_SET` | ダミーカードの選択と伏せ。 |
| `PHASE_2: EFFECT_SET` | 効果カードの選択と公開。 |
| `PHASE_3: ABILITY_SET` | 能力カードの選択と処理（覗き見による手札の強制公開など）。 |
| `PHASE_4: COMBAT_SET` | 勝負カードの選択と伏せ。 |
| `PHASE_5: REVEAL_COMBAT` | 勝負カードの一斉公開と、じゃんけんの勝敗・点数計算。 |
| `PHASE_6: GUESS_RESOLVE`| 探偵（Guess）能力の判定と点数計算。 |
| `TURN_END` | カードの回収と捨て札処理。次のターンへ。 |

※ 6ターン終了後に最終的な勝敗を決定し、同点の場合はサドンデスへ移行します。

---

## 4. データ構造・ペイロード

サーバーとクライアント間でやり取りされる通信メッセージ（プレイヤーの入力データや、サーバーからの画面更新通知など）に関するデータ構造は、実際の開発時に適切に定義を行うものとします。
